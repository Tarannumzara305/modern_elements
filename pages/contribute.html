    How to Contribute

    • Report Bugs: Found a bug? Open an issue on our GitHub repository. Provide as much detail as possible, including steps to reproduce the issue, expected behavior, and screenshots if applicable.
    • Suggest Features: Have an idea for a new feature? We’d love to hear about it. Open an issue on our GitHub repository, and clearly describe the feature and its potential benefits.
    • Improve Documentation: Good documentation helps everyone. If you notice gaps or errors in our docs, please submit a pull request with your improvements.
    • Submit Pull Requests: Fork our repository, make your changes, and submit a pull request. Ensure your code follows our contribution guidelines and is well-documented.
    • Code Reviews: Review existing pull requests. Constructive feedback helps us maintain a high-quality codebase.

    Contributing via GitHub

    1. Fork the Repository: Go to our GitHub repository and click on the "Fork" button to create your own copy.
    2. Clone the Repository: Clone your fork to your local machine using git clone: https://github.com/Tarannumzara305/modern_elements
    3. Create a Branch: Create a new branch for your work: git checkout -b feature-or-bugfix-description.
    4. Make Changes: Implement your changes in your local repository.
    5. Commit Changes: Commit your changes with a descriptive message: git commit -m "Description of your changes".
    6. Push Changes: Push your changes to your forked repository: git push origin feature-or-bugfix-description.
    7. Open a Pull Request: Navigate to the original repository and click the "New Pull Request" button. Provide a clear description of your changes and reference any related issues.
    8. Review Process: Your pull request will be reviewed by our maintainers. Be prepared to make any necessary adjustments based on their feedback.

    We appreciate every contribution, and we're excited to see how your work helps enhance our component library!
